# SubRadar Max — 阶段验收清单（战略定稿版）

本文档与战略对齐：**海外美元区、美国首发、中文可选、大陆后置**。用于产品/合规/上架的分阶段验收；完成项打勾即可。

---

## 已定战略摘要

| 维度 | 决策 |
|------|------|
| 首发国家 | **美国**（商店与运营主战场） |
| 第二波 | **加拿大**（评估后 2～6 周内） |
| 澳新 | **单独阶段**，需先验证 Plaid 与本地银行体验 |
| 货币与定价 | **USD**；月付 **$5.99～$7.99**，年付 **$39.99～$49.99**；可限时首年 **$29.99～$34.99** |
| 语言 | 默认 **英语**，**简体中文**切换（可选繁体后续） |
| 端 | **Expo 移动** + **Next.js Web** + **自建 API** |
| 银行连接 | **Plaid**，首发以 **美国机构** 为主；应用内须写明能力边界 |
| 计费 | **RevenueCat**（推荐）或等价方案 + **IAP** + **Web Stripe**；权益统一 |
| 大陆 | **海外跑通后再做**，不并行抢首发 |

## Phase 0 — 工程与协作基线（上线前必达）

**目标**：一人团队可重复发布、可观测、可回滚。

- [ ] 代码仓库：`subradar-expo` 为移动端真源；`subradar-api`、`subradar-web` 目录或独立仓库已建立（或已列入计划并有分支策略）。
- [ ] 环境变量与密钥：开发/预发/生产分离；`.env.example` 无敏感信息；Plaid、Stripe、RevenueCat、数据库等密钥不入库。
- [ ] 移动端：`eas.json` 构建配置可用；iOS/Android 版本号与构建号规则文档化（见 README）。
- [ ] 崩溃与指标：至少接入 **Sentry** 或等价（移动 + API）；核心转化漏斗事件定义（注册、连银行、Pro 购买）。
- [ ] 隐私与数据：隐私政策 URL 可访问；账户删除流程有入口（或明确计划与截止日期）。

**验收**：能从零跑通「安装 → 核心路径 → 生产构建」无阻塞文档缺口。

## Phase 1 — 美国 MVP（首发）

### 1.1 产品功能（必须）

- [ ] **手动订阅**：名称、金额、周期、下次续费日、币种 USD。
- [ ] **本地/服务端提醒**：续费前 N 天推送；时区正确（美国常用时区）。
- [ ] **Plaid（美国）**：连接只读账户；交易拉取；**recurring 检测 v1**（规则或 Plaid 信号 + 自研合并）。
- [ ] **合并策略**：自动发现与手动条目去重、合并规则清晰（文档 1 页）。
- [ ] **取消减负**：头部服务商 **深链/步骤**（可配置表）；非「代取消」须文案合规。
- [ ] **免费/Pro 边界**：免费上限或功能与付费墙一致（当前：免费最多 3 条等，与 `README` 对齐或更新 README）。
- [ ] **中文**：界面与关键文案有中英文；法律/银行说明以英文为准、中文辅助。

### 1.2 Web（MVP）

- [ ] **登录**与移动同一账户体系（Clerk / Supabase Auth 等）。
- [ ] **订阅总览**、批量编辑或导出 **CSV**（Pro）。
- [ ] **隐私政策 / 服务条款** 链接。

### 1.3 API 与安全

- [ ] Plaid `public_token` 仅在服务端换 `access_token`；不落库完整卡号。
- [ ] 同步任务队列或定时拉取；失败重试与日志脱敏。
- [ ] HTTPS 全站；CORS 与移动 deep link 配置正确。

### 1.4 合规（美国首发）

- [ ] 隐私政策：收集项（交易元数据、邮箱、设备）、用途、第三方（Plaid、分析、RevenueCat）。
- [ ] 应用内说明：**银行连接目前面向美国金融机构**；非美国用户可见不可连或引导说明。
- [ ] App Store / Google Play：**订阅类、金融工具** 所需声明与问卷如实填写（见 `app-store-connect-required.md` 并随版本更新）。

### 1.5 上架范围

- [ ] **iOS**：首发 **美国区**（或美国为主；若全球上架，应用内限制或说明银行区域）。
- [ ] **Android**：Play 上架策略与文案、**数据安全表单** 完成。
- [ ] **IAP（仅 iOS）**：`subradarpro_monthly` / `subradarpro_yearly` 与 App Store Connect 一致；基准价 **$3.99/月**、**$39.99/年**（美区）；**TestFlight** 实机验证购买与恢复。

### 1.6 成功指标（建议首 8～12 周观测）

| 指标 | 说明 |
|------|------|
| 激活后 D7 留存 | 产品是否「被记住」 |
| Plaid 连接率 | 核心差异化是否被用上 |
| 免费 → Pro 转化 | 付费墙与价值是否成立 |
| 订阅续费率 | 定价与价值是否可持续 |

**Phase 1 验收**：美国用户可完成「连银行 → 看见 recurring → 被提醒 → 找到取消路径」；Pro 可购买且权益在 **移动 + Web** 一致；商店审核无因合规/误导被拒。

## Phase 2 — 加拿大扩展

**前置**：Phase 1 美国指标稳定（无大面积「连不上」客诉）。

- [ ] Plaid **加拿大** 机构列表与联调通过。
- [ ] 商店：加拿大区上架或开放购买；文案写明支持国家。
- [ ] 定价：仍 **USD** 或按商店政策配置加元等价（二选一并全团队统一）。
- [ ] 客服与帮助文档更新 FAQ。

**验收**：加拿大用户连接主流银行成功率达到与美国同级（抽样自测 + 早期用户）。

**前置**：Plaid（或选定聚合方）对目标银行覆盖已验证；独立里程碑，不与 1.0 混评。

- [ ] 技术验证清单（银行样本、失败日志）。
- [ ] 应用商店与本地化（英语即可）。
- [ ] 支持 SLA 与文档。

## Phase 4 — 中国大陆（后置）

**前置**：海外付费与留存模型成立；有本地化运营或合作资源。

- [ ] 数据源：**短信 / 截图 OCR / 手动** 为主；微信支付宝站内数据须合规评估。
- [ ] **服务商取消攻略** 运营体系（重内容）。
- [ ] 定价：**人民币年费为主**（如 ¥68～¥128 档试探）；支付与 IAP 合规。
- [ ] 应用备案、隐私与文案（避免理财投顾表述）。

## 风险与回滚

- **Plaid 下架/机构变更**：保留手动录入为永远可用；自动发现为增强项。
- **商店拒审**：准备审核账号、测试步骤、沙盒订阅说明；**不夸大「代取消」**。
- **一人瓶颈**：Phase 1 功能砍到「能验证付费」为止，Web 可先导出 + 只读仪表盘。
